REJD — Professor Demo Script

UML Class & Sequence Diagram Generator · Eclipse Plugin · Spring 2026

This document walks through concrete demo scenarios using the REJD project's own source packages. Each scenario is chosen to highlight a specific UML concept or plugin feature.

Setup Checklist (Before You Start)

Step 1 — Open Eclipse Window 1 (developer IDE). Confirm the plugin project is imported.
Step 2 — Run As → Eclipse Application to launch Eclipse Window 2 (runtime IDE).
Step 3 — In Window 2: File → Import → General → Existing Projects → browse to ~/eclipse-workspace/rejd → Finish.
Step 4 — The REJD Diagrams tab appears automatically. Log in with any username when prompted.

Right-click any package in the Project Explorer → REJD Diagrams → Generate Class Diagram.

Demo 6 · Scope Filtering — Entire Project Most Impressive

What to right-click: The top-level rejd project in Project Explorer

Effect: Loads all ~35 classes across all packages, renders the full system diagram.

  • Use the Scope dropdown in the REJD Diagrams panel to filter by package on the fly
  • Show gwu.rejd.model scope → only model classes and their relationships
  • Show gwu.rejd.gui scope → only GUI layer
  • Switch to Entire Project → full diagram returns
💡 Talking point: "The scope filter is dynamic — no re-parsing, the model is built once and filtered in memory."

Part 2 — UML Sequence Diagram Demos

Right-click a .java file → REJD Diagrams → Generate Sequence Diagram → pick a method from the list.

Seq Demo 1 · Simple Method Chain — NoteRepository.java → save()

How: Right-click NoteRepository.java → Generate Sequence Diagram → select save(Path, List)

Concepts shown:

  • Linear message sequence (caller → repository → file system)
  • Method parameters shown on arrows
  • Return values on reply arrows
💡 Good first demo — clean, short, easy to follow.

Seq Demo 2 · Conditional / Branching — ClassDiagramController.java → setScope()

How: Right-click ClassDiagramController.java → Generate Sequence Diagram → select setScope(DiagramScope)

Concepts shown:

  • Alt fragment — different rendering paths depending on scope type
  • Self-call (controller calls its own renderCurrentScope())
  • Object creation shown as constructor call arrow

Seq Demo 3 · Multi-participant Pipeline — MultiFileProjectLoader.java → loadProject()

How: Right-click MultiFileProjectLoader.java → Generate Sequence Diagram → select loadProject(String, List)

Concepts shown:

  • Multiple lifelines — Loader, ASTParser, RelationshipExtractor, ProjectModel all appear
  • Loop fragment — iterates over each .java file
  • Object creation: new ProjectModel() shown mid-sequence
💡 Most complex sequence demo — shows the full parsing pipeline in one diagram.

Seq Demo 4 · Recursive / Nested Calls — ReplyModel.java or NotesBridge.java

How: Right-click ClassDiagramView.java → Generate Sequence Diagram → select saveReply() method via the NotesBridge inner class

Concepts shown:

  • Recursive self-call — findReplyById() calls itself for nested replies
  • Inner class as a separate lifeline

Part 3 — Plugin-Specific Feature Demos

Feature A · Persistent Notes on Diagram Nodes

  1. Generate the class diagram for gwu.rejd.model
  2. Right-click the TypeModel node in the diagram → Add Note
  3. Type a note (e.g. "Central model class — aggregates fields, methods, relationships")
  4. Click Save — note badge appears on the node
  5. Close and re-open the diagram → note is still there (persisted to .rejd/rejd-notes.json)
💡 Talking point: "Notes survive across sessions — stored as JSON in the project root, not in the diagram file."

Feature B · Export PNG

  1. With any class diagram rendered, click Export PNG in the top-left of the REJD panel
  2. Choose "Export with notes" or "Export without notes"
  3. Save to Desktop — open the file to show a clean PNG export

Feature C · Login & Author Attribution

  1. Show the login dialog that appears on Eclipse startup
  2. Enter your name → notes you create are attributed to you
  3. Notes from different users show different author names in the bubble

Feature D · Dynamic Scope Filter (Class Diagram)

  1. Right-click the project → Generate Class Diagram (loads all packages)
  2. Use the Scope dropdown at the top of the REJD panel
  3. Switch between Entire Project, gwu.rejd.model, gwu.rejd.gui
  4. Show the diagram re-renders instantly — no re-parsing
